## ЭРГО (Электроны, Рентген, Генерация)

Препринт статьи 1995 года, "Моделирование электронной эмиссии из конструкционных материалов под действием рентгеновского излучения"

**Авторы:** В.М. Лоборев (впоследствии академик РАН), А.В. Острик, В.Е. Фортов (впоследствии академик РАН, президент РАН), + соавторы (расчётная часть и инженерная методика — мои).

Статья кратко излагает результаты моей кандидатской диссертации (защищена в том же году).

## О чём работа

Задача: рассчитать энергетические, угловые и энерго-угловые распределения электронов, выбиваемых рентгеновским излучением из материалов.

**Проблема на тот момент:**
- Прямой расчёт методом Монте-Карло требовал часов (иногда дней) на мейнфрейме даже для простых веществ.
- Для инженерных задач (особенно с меняющимся во времени спектром рентгеновского излучения) это было неприемлемо.

**Что сделано:**
1. Проведены серии расчётов методом Монте-Карло на ЕС 1060 (несколько месяцев непрерывного счёта).
2. Получен массив распределений для широкого диапазона атомных номеров и начальных энергий электронов.
3. На основе этих данных подобраны **инженерные формулы**, аппроксимирующие результаты Монте-Карло.
4. Формулы позволяют получить результат для любого гомогенного материала и любого спектра рентгеновского излучения **мгновенно** (по меркам того времени).
5. Точность оказалась выше, чем у медленных расчётов зарубежных авторов (подтверждено сравнением с экспериментальными данными).
6. Дополнительно разработан гибридный подход для гетерогенных материалов — комбинация инженерных формул и Монте-Карло. Аналогов в доступной литературе на тот момент не встречалось.

## Структура модели

Модель строится так:

- Берётся **теоретическое распределение** электронов по энергии (известная аналитическая формула, выведенная при грубых допущениях).
- К нему добавляется **поправочный полином 4-й степени** от энергии электрона.
- Коэффициенты этого полинома, в свою очередь, зависят от:
  - начальной энергии электрона (полином 2-й степени)
  - эффективного атомного номера материала (полином 2-й степени)
- Поправки рассчитываются **отдельно** для трёх типов электронов:
  - фотоэлектроны
  - комптоновские электроны
  - оже-электроны
- Итоговый спектр — **взвешенная сумма** по всем типам электронов, по атомным уровням компонентов материала и по спектру падающего рентгеновского излучения.

## Что осталось за кадром (но сформировало подход к работе)

В процессе этой работы сложились некоторые подходы и методы, которые позже пригодились в других задачах — включая текущую работу с искусстевенным интеллектом.

### 1. Автоматизация пайплайна (до появления термина)

На PC 286 с DR DOS (псевдомногозадачность) в редакторе MultiEdit был настроен рабочий процесс, позволявший не выходя из редактора:

- править код (Фортран)
- отправлять задачу на мейнфрейм (VAX)
- забирать результаты
- переформатировать вывод
- автоматически строить графики в Grapher (API не было — пришлось разобраться в структуре его бинарных файлов настроек и генерировать их напрямую)
- конвертировать графику в DXF для печати на плоттере (вместо PowerPoint, которого тогда не было)

### 2. Инструментарий для пользователей

Для передачи методики другим пользователям на Фортране была написана библиотека программ **ЭРГО**, которая:

- давала интерфейс в стиле Norton Commander (два смежных окна со сменными панелями для ввода данных, конфигураций расчетов, выывода результатов в текстовой и графической форме, настроек интерфейса)
- строила линейные графики (две версии: для PC с графическим экраном и для VAX с алфавитно-цифровым терминалом — там графики рисовались символами)
- использовала двухслойную архитектуру: верхний слой (логика) не знал деталей реализации графики в нижнем слое
- была хорошо документирована. Насколько мне извествно, она применялась и для других проектов, уже без меня.

Некоторые идеи реализации позже обнаружились в Windows 3.1 — к тому моменту я к ним уже пришёл самостоятельно.

### 3. Работа с данными

- Таблицы атомных уровней оцифровывались с помощью FineReader (один из первых опытов использования ИИ-инструментов — тогда он так не назывался, но задача была та же: распознать и структурировать текст).
- Подбор коэффициентов полиномов — по сути, минимизация функции ошибки (суммы квадратов расхождений между формулой и Монте-Карло). Конкретный метод оптимизации уже не вспомню, но задача ставилась и решалась как поиск минимума loss-функции.

## Что из этого перекликается с современными подходами в ИИ

Если посмотреть на эту работу с сегодняшней точки зрения, её математическая суть — обучение параметрической модели на синтетическом наборе данных:

Данные: Монте-Карло расчёты (синтетический датасет, сгенерированный физической моделью).

Модель: параметрическая — набор полиномов с коэффициентами, зависящими от входных параметров (энергия, атомный номер).

Обучение: подбор коэффициентов минимизацией ошибки (loss) относительно обучающих данных.

Обобщение: модель работает для любых материалов и энергий в пределах области обучения.
